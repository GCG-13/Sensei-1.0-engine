# Architectural Logic: Sensei-1.0 Engine
## Deterministic Inference vs. Probabilistic Models (LLMs)

### 1. Vision General
El **Sensei-1.0 Engine** es un motor de Inteligencia Artificial desarrollado bajo el paradigma de **IA Simbólica (Symbolic AI)**. A diferencia de los modelos de lenguaje modernos (LLMs) que operan bajo probabilidades estadísticas, Sensei-1.0 utiliza un **motor de inferencia determinista** basado en reglas lógicas estructuradas para el dominio del Karate-do.

### 2. ¿Por qué IA Simbólica en lugar de Probabilística?
En la enseñanza técnica de las artes marciales, la precisión es crítica. Un modelo probabilístico (como GPT-4) puede "alucinar" o aproximar movimientos. Sensei-1.0 elimina la incertidumbre mediante:

* **Trazabilidad Total (Explainable AI - XAI):** Cada respuesta del motor puede ser rastreada hasta la regla lógica exacta que la generó.
* **Cero Alucinaciones:** El sistema no genera información nueva basada en predicciones, sino que recupera y aplica conocimiento experto validado.
* **Baja Latencia y Privacidad:** Ejecución 100% local sin necesidad de procesamiento en la nube, permitiendo una respuesta instantánea.

### 3. Componentes del Core Algorítmico

#### A. Motor de Inferencia (The Logic Brain)
El motor evalúa las entradas del usuario comparándolas con una base de conocimiento jerárquica. Utiliza una arquitectura modular que separa los datos (técnicas de karate) de la lógica de procesamiento.

#### B. Procesamiento Semántico Local
Para entender la intención del usuario sin usar modelos pesados, Sensei-1.0 implementa:
* **Fuzzy Search & Levenshtein Distance:** Permite que el sistema sea tolerante a errores ortográficos o variaciones en la nomenclatura japonesa de las técnicas.
* **Mapping de Tokens:** Un sistema de pesaje (scoring) que identifica palabras clave dentro de una cadena para determinar la categoría técnica solicitada.

### 4. Flujo de Datos (The Pipeline)
1.  **Input Capture:** Recepción de la consulta técnica.
2.  **Normalization:** Limpieza de strings y tokenización.
3.  **Semantic Matching:** Comparación contra la base de datos de conocimiento experto.
4.  **Deterministic Scoring:** Evaluación de la relevancia mediante reglas de negocio predefinidas.
5.  **Output Generation:** Respuesta técnica precisa y estructurada.

### 5. Conclusión de Ingeniería
Sensei-1.0 no busca reemplazar la conversación humana, sino servir como un **oráculo técnico infalible**. La combinación de principios **SOLID** en el código y una arquitectura de **Caja Blanca** asegura que el conocimiento del Karate-do se preserve de manera técnica y estructurada para las futuras generaciones de practicantes.
